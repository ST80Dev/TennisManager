DESCRIZIONE DEL PROGETTO
═══════════════════════════════════════════════════════════

Gioco browser single-file (HTML + React inline + Babel) di gestione carriera
tennistica ATP. Nessun server, nessuna dipendenza esterna, file unico.
Salvataggio: localStorage (chiave "atp_career_v3").
File attivo: ATP_Career-4.html (~7500+ righe)

═══════════════════════════════════════════════════════════
DECISIONI DI DESIGN
═══════════════════════════════════════════════════════════

- Filtri calendario: due toggle indipendenti combinabili (Accessibili + Mia zona)
- Rank display: sempre esatto (#420 ecc.), solo "FC" se <5pts ATP
- Stats e Forza per superficie: in schermata Profilo (non Calendario)
- Storico partite: raggruppato per torneo, card collassabili, cap 30 tornei
- Pool nomi: 20 per nazione (era 10) per ridurre collisioni fascia Challenger/ITF
- Entry fee: graduale dal basso (30/80/300/1000/2700/6000/10000)
  break-even T1 ATP250+ sempre positivo (coerente col circuito reale)
- Gain allenamento +14-17%: top 10 raggiungibile a 25-26 anni con scelte ottimali
- TrainBonus coach +15%: base 1.61 / esperto 1.90 / elite 2.24
- personalLevel NPC: floor×8/ceiling÷3 assoluti di carriera (non annuali)
  → Alcaraz può uscire stabilmente dai top 3 dopo anni difficili (Volatile)
- NPC_FIXED_POOL costante in Phase 3b: evita compressione strutturale dei top
- Noise NPC ridotto con Pacchetto B: personalLevel gestisce mobilità strutturale
- Anno 1 classifica NPC quasi statica: drift scatta a isNewYear (settimana 1 anno 2)
